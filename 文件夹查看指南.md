## 各子文件夹功能描述

### 1. code/ - 代码文件夹

**功能说明**: 存放所有Python脚本和代码文件

**文件清单**:

| 文件名 | 功能描述 |
|--------|---------|
| `data_processor.py` | 完整交互模式数据处理脚本，含可视化和暂停确认 |
| `data_processor_fast.py` | 快速处理模式数据处理脚本 |
| `final_processor.py` | 主数据处理脚本（快速模式） |
| `run_processor.py` | 模式选择入口脚本 |
| `check_and_topsis.py` | 文件检查与TOPSIS综合评价脚本 |
| `entropy_weight_fix.py` | 熵权法权重计算脚本 |
| `comprehensive_file_check.py` | 文件完整性检查脚本 |
| `comprehensive_audit.py` | 全面审计脚本 |
| `analyze_files.py` | 文件分析脚本 |
| `debug_entropy.py` | 熵权法调试脚本 |
| `organize_project.py` | 项目整理脚本 |
| `reorganize_project.py` | 项目重新整理脚本 |

**使用说明**:
- 运行 `python run_processor.py` 选择处理模式
- 运行 `python final_processor.py` 执行快速数据处理
- 运行 `python check_and_topsis.py` 执行TOPSIS综合评价

## 文件命名规范

### 1. 数据文件命名规范

**格式**: `{数据主题}_{类型}_{时间戳}.{扩展名}`

**示例**:
- 原始数据: `居民消费水平数据.xlsx`
- 处理后数据: `居民消费水平数据_已处理_20260415_081527.xlsx`
- CSV格式: `居民消费水平数据_已处理_20260415_081527.csv`
- TOPSIS分析: `居民消费水平数据_20260415_081527_TOPSIS分析.xlsx`

**时间戳格式**: `YYYYMMDD_HHMMSS`

### 2. 代码文件命名规范

**格式**: `{功能描述}.py`

**示例**:
- `data_processor.py` - 数据处理器
- `entropy_weight_fix.py` - 熵权法修复
- `comprehensive_audit.py` - 全面审计

### 3. 文档文件命名规范

**格式**: `{文档主题}.md`

**示例**:
- `文件夹查看指南.md`
- `使用说明.md`
- `项目整理总结.md`

### 4. 报告文件命名规范

**格式**: `{报告类型}_报告.{扩展名}`

**示例**:
- `数据字典.json`
- `熵权法权重分析报告.md`
- `TOPSIS综合评价报告.json`

## 可扩展性设计

### 1. 新增数据类型

如需新增数据类型，按以下步骤操作：

1. 创建新的数据文件夹，例如: `新数据类型/`
2. 在新文件夹中放入原始Excel数据文件
3. 按照现有文件命名规范重命名文件
4. 运行数据处理脚本处理新数据
5. 更新本文档，添加新文件夹的说明

### 2. 新增处理脚本

如需新增处理脚本：

1. 在 `code/` 文件夹中创建新的Python脚本
2. 遵循现有代码风格和命名规范
3. 在脚本头部添加功能说明注释
4. 更新本文档的code/文件夹文件清单

### 3. 新增分析方法

如需新增分析方法：

1. 在 `code/` 文件夹中创建新的分析脚本
2. 或扩展现有的分析脚本
3. 将分析结果保存到 `reports/` 文件夹
4. 更新本文档，添加新分析方法的说明

## 数据处理流程说明

### 阶段1: 数据清洗

1. 加载原始Excel数据
2. 格式转换（横向表格转纵向）
3. 缺失值处理（中位数填充）
4. 重复值移除
5. 异常值处理（IQR方法）
6. 格式标准化

### 阶段2: 数据标准化

1. 极差标准化（Min-Max）
2. 转换至[0,1]区间
3. 保留原始数据列
4. 新增标准化列

### 阶段3: 综合评价

1. 熵权法计算指标权重
2. TOPSIS综合评价
3. 生成分析报告

- 原始数据文件不会被修改

### 2. 版本控制

- 建议使用Git进行版本控制
- 创建 `.gitignore` 文件

### 3. 数据安全

### 4. 性能优化

- 建议使用快速模式进行日常处理

## 常见问题

### Q: 如何重新处理所有数据？

A: 运行 `python code/final_processor.py` 即可重新处理所有数据。

### Q: 处理后的文件在哪里？

A: 处理后的文件保存在原始数据所在的同一文件夹中，带有"已处理"标识。

### Q: 如何添加新的数据文件？

A: 将新的Excel文件放入对应的数据文件夹，按照命名规范重命名，然后运行数据处理脚本。

### Q: 分析报告如何查看？

A: 分析报告有JSON和Markdown两种格式，Markdown格式可用任何Markdown编辑器查看。
