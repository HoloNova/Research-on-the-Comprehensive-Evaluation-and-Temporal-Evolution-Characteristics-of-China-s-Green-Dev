# 仓库提交指南

## 提交前检查清单

在提交代码前，请确保完成以下检查：

- [ ] 所有代码文件已测试通过
- [ ] 文档已更新（README.md、文件夹查看指南.md）
- [ ] 没有临时文件或调试代码
- [ ] .gitignore已正确配置
- [ ] 敏感信息已移除

---

## 初始仓库设置

### 1. 初始化Git仓库

```bash
# 进入项目目录
cd 统计建模

# 初始化Git仓库
git init
```

### 2. 配置用户信息

```bash
# 配置用户名
git config user.name "您的用户名"

# 配置邮箱
git config user.email "您的邮箱"
```

### 3. 添加文件到暂存区

```bash
# 添加所有文件
git add .

# 或选择性添加
git add code/
git add docs/
git add *.md
git add .gitignore
```

### 4. 创建初始提交

```bash
# 创建提交
git commit -m "初始提交：项目基础结构和数据处理脚本

- 添加数据处理脚本（data_processor系列）
- 添加综合评价脚本（熵权法、TOPSIS）
- 添加文档和使用说明
- 添加原始数据和处理后数据
- 配置.gitignore文件"
```

### 5. 关联远程仓库（可选）

```bash
# 添加远程仓库
git remote add origin <远程仓库URL>

# 推送到远程仓库
git push -u origin main
# 或
git push -u origin master
```

## .gitignore说明

当前.gitignore文件已配置忽略以下内容：

### Python相关
- `__pycache__/` - Python缓存目录
- `*.py[cod]` - 编译的Python文件
- `*.egg-info/` - Egg信息目录
- `*.egg` - Egg包

### 虚拟环境
- `venv/`
- `ENV/`
- `env/`

### IDE相关
- `.vscode/` - VS Code配置
- `.idea/` - IntelliJ IDEA配置
- `*.swp` - Vim交换文件
- `*.swo` - Vim交换文件
- `*~` - 备份文件

### 操作系统
- `.DS_Store` - macOS系统文件
- `Thumbs.db` - Windows缩略图

### 临时文件
- `*.tmp` - 临时文件
- `*.bak` - 备份文件
- `*.log` - 日志文件

### 数据文件（可选）
如不需要提交数据文件，可取消以下注释：
```
# *.xlsx
# *.csv
# 国民消费水平/
# 科技/
# 能源/
# 资源与环境/
```

## 分支管理建议

### 主分支
- `main` 或 `master` - 稳定版本，用于发布

### 开发分支
- `develop` - 开发分支，集成新功能

### 功能分支
- `feature/xxx` - 新功能开发
- `fix/xxx` - 问题修复

## 提交信息规范

### 提交信息格式

```
<类型>(<范围>): <主题>

<正文>

<页脚>
```

### 类型说明

| 类型 | 说明 |
|------|------|
| `feat` | 新功能 |
| `fix` | 修复bug |
| `docs` | 文档更新 |
| `style` | 代码格式调整 |
| `refactor` | 重构 |
| `perf` | 性能优化 |
| `test` | 测试相关 |
| `chore` | 构建/工具相关 |

### 提交信息示例

```
feat: 添加数据可视化功能

- 添加matplotlib图表生成
- 支持多种图表类型
- 添加图表导出功能

Closes #12
```

```
fix: 修复熵权法计算中的NaN问题

- 处理空值情况
- 添加数据验证
- 更新测试用例

Fixes #45
```

```
docs: 更新文件夹查看指南

- 添加新数据类型说明
- 更新命名规范
- 添加常见问题解答
```

## 数据文件管理建议

### 选项1: 提交所有数据

如果数据文件较小且需要版本控制：
- 保留当前.gitignore配置
- 所有数据文件都会被提交

### 选项2: 只提交原始数据

如果只需要版本控制原始数据：
- 在.gitignore中添加：
```
*_已处理_*.xlsx
*_已处理_*.csv
*_TOPSIS分析.xlsx
```

### 选项3: 不提交数据

如果数据文件较大：
- 取消.gitignore中数据部分的注释
- 使用外部存储（如Git LFS、云存储等）
- 提供数据下载脚本或说明

## 常见问题

### Q: 如何修改最后一次提交？

```bash
# 修改提交信息
git commit --amend

# 添加遗漏的文件
git add <遗漏的文件>
git commit --amend
```

### Q: 如何撤销提交？

```bash
# 撤销最近一次提交，但保留更改
git reset --soft HEAD~1

# 撤销最近一次提交，并丢弃更改
git reset --hard HEAD~1
```

### Q: 如何忽略已追踪的文件？

```bash
# 从追踪中移除，但保留本地文件
git rm --cached <文件名>
git commit -m "停止追踪<文件名>"
```

## 推荐工作流

1. **创建功能分支**
   ```bash
   git checkout -b feature/new-feature
   ```

2. **开发并提交**
   ```bash
   git add .
   git commit -m "feat: 添加新功能"
   ```

3. **合并到开发分支**
   ```bash
   git checkout develop
   git merge feature/new-feature
   ```

4. **发布到主分支**
   ```bash
   git checkout main
   git merge develop
   git tag v1.0.0
   git push origin main --tags
   ```
